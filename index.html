Propuesta UX/UI — TWINS DESPACHO

Análisis del código existente + wireframes con mejoras concretas para cada pantalla.

🔑

Login más seguro

Reemplazar clic simple por avatares con PIN numérico visual para el usuario ADMIN.

  • Avatares con iniciales de color
  • PIN visual tipo puntos
  • Indicador de rol activo
📦

Carga de Ruta mejorada

Agregar barra de progreso visual y resumen en tiempo real del manifesto.

  • Contador de cajas en vivo
  • Barra de progreso objetivo
  • Feedback visual al agregar
✅

Confirmación mejorada

Modal con resumen de datos antes de guardar: evita errores críticos.

  • Muestra placa, chofer, total
  • Botones claros HOY / MAÑANA
  • Sin confirmaciones accidentales
📋

Historial con filtros

Chips de filtro rápido y acciones directas en cada tarjeta (WA + PDF).

  • Filtros por día / semana / mes
  • Acceso directo a WA
  • Exportar PDF por ruta
🚛

Flota + alertas SOAT

Indicadores de color claros para días restantes de SOAT con umbral de alerta.

  • Verde / Naranja / Rojo
  • Icono de camión por placa
  • Fecha exacta visible
🗂️

Navegación inferior

Reordenar nav: Cargar → Data → Flota → PDF. Etiquetas más cortas y claras.

  • Indicador activo más visible
  • Orden lógico de flujo
  • PDF oculto si no es ADMIN

📌 Flujo de uso actual vs. propuesto

Login → Seleccionar placa/chofer/aux → Agregar clientes + cajas → ★ Confirmar con resumen → Historial / Compartir WA

Avatares con color de identidad

Cada usuario tiene un color único. Más rápido de identificar visualmente, reduce errores de selección en condiciones de trabajo rápido.

2

Eliminar prompt() nativo del browser

El prompt() actual bloquea el hilo principal y no funciona bien en algunos navegadores móviles. Reemplazarlo con un campo PIN visual mejora la UX y la compatibilidad.

Barra de progreso de cajas

Permite definir un objetivo de cajas por ruta y visualizar el avance. Útil para saber si falta cargar antes de salir.

1

Total siempre visible

El contador de cajas aparece pegado al fondo de la lista, visible sin hacer scroll. Reduce errores de conteo.

2

Campo de Factura opcional

Agregar un segundo campo de texto para número de factura junto al cliente, útil para trazabilidad.

3

Validación de campos vacíos

Actualmente el botón ADD falla en silencio si falta la placa. Propuesta: resaltar en rojo el campo faltante y mostrar mensaje claro.
